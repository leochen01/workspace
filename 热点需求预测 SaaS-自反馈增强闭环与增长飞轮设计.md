## 一、核心洞察：为什么需要自反馈闭环

### 1.1 当前模型的局限性

| 问题 | 现状 | 风险 |
|------|------|------|
| **权重来源** | 专家经验设定 (need_taxonomy.json / weight_matrix.json) | 未经验证，可能与真实需求脱节 |
| **校准方式** | 人工调参 | 无法规模化，响应慢 |
| **预测验证** | 无闭环验证 | 无法衡量准确率，无法迭代 |
| **人群差异** | 8 类人群固定权重 | 无法适配细分行业/地区差异 |

### 1.2 自反馈闭环的价值

```
传统模式:
专家设定权重 → 模型预测 → 输出结果 → (结束)
                    ↓
              无法验证准确性

自反馈模式:
专家设定权重 → 模型预测 → 用户反馈 → 权重校准 → 更准预测 → 更多用户
                    ↑                                        │
                    └────────────────────────────────────────┘
                         形成增强闭环
```

```python
def calculate_need_score_adjustment(need_id, event_type, segment_id):
    """
    计算某个需求在特定事件类型和人群下的分数调整值
    """
    feedbacks = db.feedback.find({
        "need_id": need_id,
        "event_type": event_type,
        "segment_id": segment_id,
        "created_at": {"$gte": now() - timedelta(days=30)}
    })
    
    positive_signals = 0
    negative_signals = 0
    
    for fb in feedbacks:
        if fb["type"] == "accuracy_rating" and fb["rating"] == "accurate":
            positive_signals += 1.0
        elif fb["type"] == "accuracy_rating" and fb["rating"] == "inaccurate":
            negative_signals += 1.0
        elif fb["type"] == "behavior" and fb["action"] == "favorite":
            positive_signals += 0.6
        elif fb["type"] == "behavior" and fb["action"] == "skip":
            negative_signals += 0.5
        elif fb["type"] == "behavior" and fb["action"] == "share":
            positive_signals += 0.8
    
    total_signals = positive_signals + negative_signals
    if total_signals < 10:  # 样本不足，不调整
        return 0.0
    
    net_score = (positive_signals - negative_signals) / total_signals
    adjustment = net_score * 0.1  # 最大调整幅度 10%
    
    return clamp(adjustment, -0.1, 0.1)
```

```python
async def generate_validation_report(prediction_id):
    prediction = db.predictions.get(prediction_id)
    event = prediction.event
    predicted_needs = prediction.top_needs
    
    # 收集验证数据
    validation_data = {}
    for need in predicted_needs[:5]:
        keywords = need.trigger_keywords
        trend_data = await fetch_trend_data(keywords, days=7)
        validation_data[need.id] = {
            "search_growth": trend_data.search_growth,
            "content_growth": trend_data.content_growth,
            "product_growth": trend_data.product_growth,
        }
    
    # 计算预测准确度
    accuracy_scores = []
    for need_id, data in validation_data.items():
        avg_growth = (data["search_growth"] + data["content_growth"]) / 2
        if avg_growth > 50:  # 显著增长
            accuracy_scores.append(0.9)
        elif avg_growth > 20:
            accuracy_scores.append(0.7)
        elif avg_growth > 5:
            accuracy_scores.append(0.5)
        else:
            accuracy_scores.append(0.3)
    
    overall_accuracy = sum(accuracy_scores) / len(accuracy_scores)
    
    # 生成报告
    report = {
        "prediction_id": prediction_id,
        "overall_accuracy": overall_accuracy,
        "need_validations": validation_data,
        "generated_at": datetime.now(),
    }
    
    # 发送给用户
    await send_email(
        to=prediction.user_id,
        subject=f"7 天验证报告：{event.title} 预测准确度 {overall_accuracy:.0%}",
        template="validation_report",
        data=report
    )
    
    # 存入数据库用于模型校准
    db.validation_reports.insert(report)
    
    return report
```

### 3.2 权重更新算法

```python
# 伪代码：基于反馈的权重校准

class WeightCalibrator:
    def __init__(self):
        self.learning_rate = 0.01
        self.decay_factor = 0.95  # 时间衰减
        
    def calculate_time_weight(self, feedback_time):
        """计算时间衰减权重"""
        days_elapsed = (datetime.now() - feedback_time).days
        return self.decay_factor ** days_elapsed
    
    def calculate_user_credibility(self, user_id):
        """
        计算用户可信度
        基于历史反馈的准确度和一致性
        """
        user_feedbacks = db.feedback.find({"user_id": user_id})
        if len(user_feedbacks) < 5:
            return 0.5  # 新用户，默认可信度
        
        # 计算用户历史反馈与最终验证的一致性
        consistency_score = self.calculate_consistency(user_feedbacks)
        return clamp(consistency_score, 0.2, 1.0)
    
    def update_constraint_weights(self, event_type_id, feedbacks):
        """
        更新约束层权重
        """
        current_weights = db.event_types.get(event_type_id)["constraint_weights"]
        
        # 聚合反馈信号
        signal_by_constraint = defaultdict(list)
        for fb in feedbacks:
            time_weight = self.calculate_time_weight(fb["created_at"])
            user_credibility = self.calculate_user_credibility(fb["user_id"])
            
            for constraint_id, contribution in fb["constraint_contributions"].items():
                signal = contribution * time_weight * user_credibility
                signal_by_constraint[constraint_id].append(signal)
        
        # 计算权重调整
        new_weights = {}
        for constraint_id, signals in signal_by_constraint.items():
            avg_signal = sum(signals) / len(signals)
            current = current_weights.get(constraint_id, 0.5)
            
            # 梯度下降更新
            gradient = avg_signal * self.learning_rate
            new_weight = current + gradient
            new_weights[constraint_id] = clamp(new_weight, 0.0, 1.0)
        
        return new_weights
    
    def ab_test_weights(self, new_weights, control_weights, traffic_split=0.05):
        """
        A/B 测试新权重
        """
        # 创建实验组
        experiment = db.experiments.create({
            "name": f"weight_update_{datetime.now().isoformat()}",
            "control_weights": control_weights,
            "treatment_weights": new_weights,
            "traffic_split": traffic_split,
            "status": "running",
            "started_at": datetime.now(),
        })
        
        # 监控指标
        metrics = self.monitor_experiment(experiment.id, duration_days=7)
        
        if metrics["treatment_accuracy"] > metrics["control_accuracy"] * 1.05:
            # 实验组准确度提升 5% 以上，全量发布
            self.deploy_weights(new_weights)
            return True
        else:
            # 回滚
            self.rollback_weights(control_weights)
            return False
```

```
K = 每个用户平均分享次数 × 分享转化率

目标：K > 0.5 (每 2 个用户带来 1 个新用户)

假设:
- 平均每个用户分享 2 次
- 分享点击率 15%
- 点击后注册转化率 20%

K = 2 × 0.15 × 0.20 = 0.06 (需要优化)

优化策略:
- 增加分享激励 (积分 → 现金券)
- 优化分享卡片设计 (增加点击率)
- 优化落地页 (提高转化率)
```

## 七、风险与应对

| 风险 | 概率 | 影响 | 应对方案 |
|------|------|------|----------|
| **反馈量不足** | 高 | 高 | 增加激励 (积分→现金券)，降低反馈门槛 (一键评分) |
| **恶意反馈** | 中 | 中 | 用户可信度加权，异常检测，IP 限流 |
| **权重更新导致准确度下降** | 中 | 高 | A/B 测试 + 自动回滚，小步快跑 (每次更新<5% 流量) |
| **外部数据源不可用** | 中 | 中 | 多数据源备份，降级为纯用户反馈校准 |
| **用户隐私担忧** | 低 | 高 | 明确告知数据用途，提供退出选项，数据脱敏 |

## 八、总结

### 核心设计原则

1. **反馈分层**: L1-L4 四层反馈，平衡密度与价值
2. **渐进校准**: 小步更新 + A/B 测试 + 自动回滚
3. **激励相容**: 用户获得价值 → 愿意反馈 → 模型更准 → 用户获得更多价值
4. **飞轮效应**: 每个环节都设计增强回路，而非线性增长

### 预期效果 (12 个月后)

| 指标 | 当前 | 目标 | 提升 |
|------|------|------|------|
| 模型准确度 | 专家经验 (未验证) | >85% (基于验证) | - |
| 用户留存 (D30) | - | >50% | - |
| 付费转化率 | - | >6% | - |
| 病毒系数 K | 0 | >0.3 | - |
| 新用户来自推荐 | 0% | >30% | - |
| API 收入占比 | 0% | >30% | - |

### 一句话总结

**通过四层反馈体系收集用户信号，用校准引擎自动优化模型权重，再借社交传播和 API 生态放大网络效应，最终形成"用户越多→数据越多→模型越准→用户越多"的增长飞轮。**
