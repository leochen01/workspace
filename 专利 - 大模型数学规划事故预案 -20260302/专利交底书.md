## 三、背景技术

### 3.1 技术背景

电网事故预案是电力系统安全稳定运行的重要保障措施，用于在发生故障或异常工况时指导调度人员快速、正确地处置事故，恢复电网正常运行。随着电网规模不断扩大和可再生能源高比例接入，电网运行的不确定性和复杂性显著增加，对事故预案的编制质量、响应速度和覆盖范围提出了更高要求。

### 3.2 现有技术及其不足

目前电网事故预案编制主要采用以下技术方案：

#### 3.2.1 基于最优潮流（OPF）的调度优化

**技术特点：**
- 提供满足物理约束的可行解
- 基于严格的数学模型保证方案可行性

**不足之处：**
- 不具备语义理解能力，无法理解调度规程和专家经验
- 不能自动生成多预案供调度员选择
- 无因果推理能力，无法解释方案生成逻辑
- 无事故解释能力，难以生成可读性强的操作说明

#### 3.2.2 基于规则库的专家系统

**技术特点：**
- 可自动匹配预定义预案
- 响应速度快

**不足之处：**
- 规则维护成本高，需人工持续更新
- 泛化能力弱，难以适应未见过的事故场景
- 难以适应新能源波动场景下的复杂事故

#### 3.2.3 基于大模型的智能问答系统

**技术特点：**
- 具备自然语言理解能力
- 可生成事故分析报告

**不足之处：**
- 不具备电气物理约束保证
- 输出结果可能不可执行，存在安全隐患
- 无安全性保障机制

### 3.3 现有技术的核心不足

现有技术缺乏**认知理解能力**与**严格物理约束保证能力**的融合机制，具体体现在：

1. **无法将语义推理结果转化为优化模型输入**：大模型生成的处置建议无法直接用于数学规划求解
2. **无法将优化约束冲突反馈至语义层**：数学规划求解失败时无法有效反馈给大模型进行修正
3. **缺乏双向闭环**：现有方案多为单向调用，缺乏迭代优化机制
4. **无法实现结构化自动生成与物理可行性统一**：生成的预案要么可读性差，要么物理不可行

### 3.4 需要规避的现有专利方向

- 纯 OPF 事故恢复方法
- 基于规则库的预案生成系统
- 单纯大模型生成事故分析方法
- 传统 SCADA 自动告警系统

## 四、发明目的

本发明旨在解决现有技术中存在的上述问题，提供一种基于大模型与数学规划融合的电网事故预案自动生成方法及系统，实现：

1. **语义理解与物理约束的融合**：将大模型的认知理解能力与数学规划的严格物理约束保证能力有机结合
2. **语义 - 数学自动映射**：实现从自然语言描述到数学规划模型的自动转换
3. **双向闭环优化**：建立大模型生成→优化校验→约束冲突提取→大模型修正的闭环机制
4. **多方案自动生成**：自动生成多套可行预案供调度员选择
5. **结构化预案输出**：自动生成符合调度规程的标准操作票和风险提示

## 五、技术方案

### 5.1 整体技术架构

本发明提出的电网事故预案自动生成方法，包括以下 7 个核心步骤：

```
┌─────────────────────────────────────────────────────────────────┐
│  步骤 1：故障场景分析与解析（大模型）                              │
│  输入：故障描述、电网运行状态、天气预报、历史故障数据               │
│  输出：结构化的故障分析报告                                        │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  步骤 2：基于大模型的事故理解与恢复目标提取                         │
│  输出：故障影响区域、风险等级、恢复优先级排序、恢复目标函数权重      │
│  关键：恢复目标以参数化形式输出（W1=0.7, W2=0.2, W3=0.1）          │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  步骤 3：语义到数学模型的自动映射机制（★核心创新）                  │
│  功能：将语义输出映射为优化目标函数、数学约束表达式、数学变量        │
│  示例："优先恢复重要负荷区域 A" → Max Σ α_i * P_i                 │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  步骤 4：数学规划模型构建与求解                                     │
│  模型类型：MILP/MISOCP/DC-OPF/SCOPF                              │
│  输出：开关状态序列、发电机出力调整、切负荷方案                     │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  步骤 5：物理仿真验证与安全校核                                     │
│  验证：AC 潮流计算、静态安全分析、N-1 校验                          │
│  若不可行 → 自动生成冲突约束列表 → 返回步骤 2 或 3（闭环）           │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  步骤 6：多方案评估与迭代优化                                       │
│  评估维度：成本、恢复率、操作复杂度                                 │

                              ↓

└─────────────────────────────────────────────────────────────────┘
```

### 5.2 核心创新点

#### 创新点 A（最核心）：语义 - 数学自动映射机制

**技术特征：**
1. 大模型输出的恢复目标与约束自动转换为数学规划变量与约束表达式
2. 形成可求解的优化模型
3. 这是本发明与现有技术的本质区别

**具体实现：**
- **目标函数映射**：将大模型生成的恢复优先级排序映射为目标函数权重系数
- **约束表达式映射**：将语义约束（如"电压不能越限"）映射为数学不等式约束
- **设备实体映射**：将设备名称（如"主变 T1"）映射为数学变量索引

**算法步骤：**
```
输入：大模型输出的语义描述 S
输出：数学规划模型 M = {目标函数 f(x), 约束 g(x)≤0, h(x)=0}

步骤 3.1：实体识别与链接
  - 从 S 中识别设备实体 E = {e1, e2, ..., en}
  - 将 E 链接到电网模型中的设备索引 I = {i1, i2, ..., in}

步骤 3.2：目标函数构建
  - 解析 S 中的恢复优先级描述
  - 生成权重向量 W = {w1, w2, ..., wm}
  - 构建目标函数 f(x) = Σ wi * fi(x)

步骤 3.3：约束表达式构建
  - 解析 S 中的约束描述
  - 映射为数学约束 g(x)≤0 或 h(x)=0
  - 添加设备操作逻辑约束

步骤 3.4：模型验证
  - 检查模型完整性
  - 输出可求解的数学规划模型 M
```

#### 创新点 B：双向闭环优化机制

**技术特征：**
- LLM 生成 → 优化校验 → 约束冲突提取 → LLM 修正
- 区别于现有技术的"单向调用"

**闭环流程：**
```
大模型生成初始方案
        ↓
数学规划求解
        ↓
物理仿真验证
        ↓
    ┌───────┐
    │ 可行？ │
    └───────┘
       │
   ┌───┴───┐
   │       │
  是       否
   │       │
   ↓       ↓
输出    提取冲突约束
预案      ↓
      反馈给大模型
        ↓
    大模型修正
        ↓
    返回重新求解
```

#### 创新点 C：多目标权重动态生成机制

**技术特征：**
- 恢复目标权重由大模型根据事故场景动态生成
- 适配不同事故场景（如重要用户保电 vs. 一般负荷恢复）

**权重生成规则：**
| 事故场景 | W_负荷恢复 | W_操作次数 | W_运行成本 |
|----------|-----------|-----------|-----------|
| 重要用户停电 | 0.8 | 0.1 | 0.1 |
| 一般线路故障 | 0.6 | 0.2 | 0.2 |
| 大面积停电 | 0.9 | 0.05 | 0.05 |
| 设备过载 | 0.5 | 0.3 | 0.2 |

#### 创新点 D（从属）：结构化预案自动生成

**技术特征：**
- 操作步骤自动排序（符合调度规程）
- 风险自动标注（基于历史案例和专家知识）
- 自动生成图表（拓扑变化图、负荷统计图）

### 实施例一：220kV 变电站主变故障预案生成

#### 8.1.1 场景描述

某 220kV 变电站主变 T1 故障跳闸，需要自动生成事故处置预案。

#### 8.1.2 输入数据

| 数据项 | 内容 |
|--------|------|
| 故障设备 | 220kV 主变 T1 |
| 负荷水平 | 85%（高温天气） |
| 当前运行方式 | 双主变并列运行 |
| 影响区域 | A 区工业负荷 |
| 天气条件 | 高温（35°C） |
| 重要用户 | A 区某化工厂（一级重要用户） |

#### 8.1.3 执行过程

**步骤 1：故障场景分析与解析**

大模型接收输入数据，输出故障分析报告：
```json
{
  "故障设备": "220kV 主变 T1",
  "影响负荷": "120MW",
  "风险等级": "高",
  "受影响区域": ["A 区"],
  "重要用户": ["某化工厂"],
  "建议优先恢复": "工业用户"
}
```

**步骤 2：恢复目标与约束提取**

大模型生成恢复目标参数：
```json
{
  "目标函数权重": {
    "W_load": 0.75,    // 负荷恢复权重（高，因为有重要用户）
    "W_switch": 0.15,  // 开关操作权重
    "W_cost": 0.10     // 运行成本权重
  },
  "恢复优先级": ["某化工厂", "A 区工业负荷", "A 区居民负荷"],
  "约束条件": [
    "电压不能越限",
    "线路不能过载",
    "开关操作次数尽量少"
  ]
}
```

**步骤 3：语义→数学自动映射**

语义映射引擎将大模型输出转换为数学规划模型：

**目标函数：**
```
min 0.75 * (未恢复负荷量) + 0.15 * (开关操作次数) + 0.10 * (运行成本)
```

**约束条件：**
```
潮流平衡约束：P_gen - P_load = P_loss
线路容量约束：|P_line| ≤ P_line_max
电压约束：V_min ≤ V ≤ V_max
开关操作逻辑约束：∑ |s_i - s_i0| ≤ N_max
```

**步骤 4：数学规划求解**

调用 CPLEX 求解器求解 MILP 模型，输出：
```json
{
  "开关操作": [
    {"设备": "联络线 L3 开关", "操作": "合闸"},
    {"设备": "T1 低压侧开关", "操作": "分闸"}
  ],
  "发电机调整": [
    {"设备": "发电机 G2", "调整": "+50MW"}
  ],
  "切负荷方案": [
    {"区域": "B 区", "负荷": "20MW", "类型": "非重要负荷"}
  ]
}
```

进行 AC 潮流计算和 N-1 校验：
```

- 线路潮流：无过载 ✓
- N-1 校验：满足 ✓
结论：方案可行
```

## 九、权利要求书（建议稿）

### 权利要求 1（独立权利要求）

一种基于大模型与数学规划融合的电网事故预案自动生成方法，其特征在于，包括以下步骤：

**步骤 S1**：接收故障信息，利用大语言模型进行故障场景分析，生成结构化的故障分析报告；

**步骤 S2**：利用所述大语言模型从所述故障分析报告中提取恢复目标和约束条件，输出参数化的恢复目标函数权重；

**步骤 S3**：通过语义 - 数学映射引擎，将所述恢复目标和约束条件自动转换为数学规划模型，包括目标函数、约束表达式和决策变量；

**步骤 S4**：调用数学规划求解器对所述数学规划模型进行求解，得到电网恢复方案；

**步骤 S5**：对所述电网恢复方案进行物理仿真验证，若验证通过则执行步骤 S6，若验证不通过则提取冲突约束并返回步骤 S2 或 S3 进行修正；

**步骤 S6**：利用所述大语言模型对所述电网恢复方案进行评估和优化，生成多套备选预案；

**步骤 S7**：利用所述大语言模型自动生成结构化的预案文档，包括调度操作票、风险提示和统计图表。

### 权利要求 2（从属权利要求）

根据权利要求 1 所述的方法，其特征在于，所述步骤 S3 中的语义 - 数学映射包括：
- 实体识别与链接：从语义描述中识别设备实体并链接到电网模型中的设备索引；
- 目标函数映射：将恢复优先级描述映射为目标函数权重系数；
- 约束表达式映射：将语义约束映射为数学不等式约束或等式约束。

### 权利要求 3（从属权利要求）

根据权利要求 1 所述的方法，其特征在于，所述步骤 S5 中的物理仿真验证包括 AC 潮流计算、静态安全分析和 N-1 校验。

### 权利要求 4（从属权利要求）

根据权利要求 1 所述的方法，其特征在于，所述步骤 S6 中的评估维度包括运行成本、负荷恢复率和操作复杂度。

### 权利要求 5（从属权利要求）

根据权利要求 1 所述的方法，其特征在于，所述数学规划模型包括混合整数线性规划 MILP、混合整数二阶锥规划 MISOCP、直流最优潮流 DC-OPF 或安全约束最优潮流 SCOPF。

### 权利要求 6（系统独立权利要求）

一种实现权利要求 1-5 任一项所述方法的电网事故预案自动生成系统，其特征在于，包括：
- 用户交互层，用于事故信息输入和预案展示；
- 大模型服务层，用于故障场景分析、恢复目标提取、语义映射、多方案评估和预案文档生成；
- 数学规划求解层，用于数学规划模型的求解；
- 物理仿真验证层，用于 AC 潮流计算、静态安全分析和 N-1 校验；
- 数据支撑层，用于存储电网模型、历史故障案例、调度规程和设备参数。

本发明公开了一种基于大模型与数学规划融合的电网事故预案自动生成方法及系统。该方法包括：利用大语言模型进行故障场景分析和恢复目标提取；通过语义 - 数学映射引擎将语义描述自动转换为数学规划模型；调用数学规划求解器求解得到电网恢复方案；进行物理仿真验证并形成双向闭环优化；自动生成多套备选预案和结构化预案文档。本发明实现了大模型的认知理解能力与数学规划的物理约束保证能力的有机融合，显著缩短了预案编制时间，提高了方案可行性和适应性。
