### 专利点 1️⃣：基于多模态向量融合的电力知识统一检索方法

| 维度 | 内容 |
|------|------|
| **技术领域** | 多模态信息检索 + 电力知识工程 |
| **核心创新** | 将电力文档、图纸、视频、表格等多模态数据映射到统一向量空间，实现跨模态语义检索 |
| **技术方案** | 1. 多模态编码器（文本/图像/表格分别编码）<br>2. 统一向量空间投影（对比学习对齐）<br>3. 混合检索（向量 + 关键词 + 电力本体）<br>4. 多模态结果融合排序 |
| **权利要求要点** | - 多模态编码器的电力领域适配（如电气图纸专用编码器）<br>- 统一向量空间的构建方法<br>- 跨模态检索的融合排序算法 |
| **专利类型** | 发明专利 |
| **创新等级** | ⭐⭐⭐⭐⭐ 核心专利 |
| **私有化适配** | 本地部署的轻量级多模态编码器（如 CLIP 蒸馏版本） |

### 专利点 2️⃣：基于角色 - 场景双维度的电力知识库动态权限控制方法

| 维度 | 内容 |
|------|------|
| **技术领域** | 数据安全 + 权限管理 |
| **核心创新** | 传统 RBAC 仅基于角色，本发明引入"场景"维度，实现动态权限控制 |
| **技术方案** | 1. 角色维度：调度员/检修员/管理员/实习生<br>2. 场景维度：日常工作/应急抢修/培训考试/对外汇报<br>3. 数据分级：公开/内部/秘密/核心秘密<br>4. 动态权限计算：Permission = f(角色，场景，数据级别，时间，位置)<br>5. 权限缓存与实时校验机制 |
| **权利要求要点** | - 角色 - 场景双维度权限模型<br>- 动态权限计算函数<br>- 权限变更的实时生效机制 |
| **专利类型** | 发明专利 |
| **创新等级** | ⭐⭐⭐⭐⭐ 核心专利 |
| **私有化适配** | 本地权限策略引擎，无需外部认证服务 |

#### 权限计算模型

```
Permission = f(Role, Scenario, DataLevel, Time, Location)

其中：
- Role ∈ {调度员，检修员，管理员，实习生，...}
- Scenario ∈ {日常工作，应急抢修，培训考试，对外汇报，...}
- DataLevel ∈ {公开，内部，秘密，核心秘密}
- Time ∈ {工作时间，非工作时间，节假日}
- Location ∈ {办公区，控制室，远程，...}

权限计算规则示例：
IF Role = "调度员" AND Scenario = "应急抢修" AND DataLevel ≤ "秘密"
   THEN Permission = "读 + 下载"
ELSE IF Role = "实习生" AND Scenario = "培训考试" AND DataLevel = "公开"
   THEN Permission = "读"
ELSE Permission = "无权限"
```

#### 场景自动识别

| 场景 | 识别特征 | 权限调整 |
|------|----------|----------|
| 日常工作 | 正常工作时间、常规查阅操作 | 标准权限 |
| 应急抢修 | 非工作时间 + 调度员 + 查阅预案/规程 | 临时提升权限 |
| 培训考试 | 培训系统入口、考试题目查阅 | 仅限公开/内部资料 |
| 对外汇报 | 对外交流时间、PPT/报告生成 | 仅限公开资料 |

#### 知识图谱 Schema

```
实体类型：
- 设备（主变/线路/开关/保护装置/...）
- 规程（调度规程/检修规程/安全规程/...）
- 故障（短路/过载/接地/...）

- 位于（设备 → 地点）
- 属于（设备 → 设备类型）
- 遵循（设备 → 规程）
- 发生（故障 → 设备）
- 涉及（案例 → 故障）
- 负责（人员 → 设备）
```

#### 异常行为定义

#### 风险评分模型

```
风险评分 = Σ(异常类型权重 × 异常次数)

- 非工作时间访问：0.15
- 异常搜索：0.1
- 频繁导出：0.25
- 异地登录：0.15

风险等级：
- 0-0.3: 低风险（记录日志）
- 0.3-0.6: 中风险（发送告警）
- 0.6-1.0: 高风险（实时阻断 + 人工审核）
```

### 专利点 1：多模态统一检索

**技术壁垒**：
- 电力图纸/表格的专用编码器训练

**权利要求布局**：
- 独立权利要求：多模态编码→统一向量空间→混合检索→融合排序
- 从属权利要求：电力图纸编码器、表格结构化提取、本体增强检索

**规避设计**：
- 区别于通用多模态检索（如 CLIP）
- 强调电力领域适配（图纸符号识别、电气术语理解）

**技术壁垒**：
- 场景的自动识别（如从用户操作推断当前场景）

**权利要求布局**：
- 独立权利要求：角色维度 + 场景维度 + 动态计算函数
